# AGENTS.md — 開発ルール

このプロジェクトは「できるだけ強い麻雀AI」を最終目標とする、Next.js + TypeScript 製のブラウザ麻雀アプリです。
コードを編集する自律エージェント (あなた) が守るべきルールをここに集約します。

## 最重要原則

1. **動く成果物を最優先する**。説明より実装。1人用練習画面が動かなくなる変更は禁止。
2. **AIの強さを優先する**。見た目より AI のロジック改善を優先。
3. **後から拡張しやすい設計**を保つ。`lib/mahjong` と `lib/ai` を分離した状態を崩さない。
4. **動作確認は必須**。`npm run build` を通し、必要に応じて `npm run dev` で挙動確認する。

## 技術構成

- Next.js 14 App Router (`src/app`)
- TypeScript strict
- React 18
- 追加ランタイムは tsx (テスト用) のみ。Node 標準 test runner を使う。
- 重い数値計算は将来 Python/WASM 実装に差し替え可能なよう、純関数を維持する。

## ディレクトリ規約

```
src/
  app/                    画面 (Next.js App Router)
  components/             UI部品
  lib/
    mahjong/              麻雀ルール、牌、シャンテン、受け入れ、点数
    ai/                   AI評価、推奨打牌、鳴き判断、リーチ判断、守備
    game/                 1人練習用ゲームステート
scripts/                  自己対戦・シミュレーション用 (今後)
python/                   学習・分析用 (今後)
test/                     Node test runner のテスト
```

新規ファイルは原則これらに収めること。`lib/mahjong` から `lib/ai` を import するのは可だが、逆は禁止。`lib/mahjong` から React に依存しないこと。

## エージェントの作業手順

1. 着手前に `PROJECT_STATUS.md`、`TODO.md` を読む
2. タスクを `TodoWrite` 等に分解する
3. 実装する
4. `npm run build` を必ず通す
5. 必要なら `npm test` (= `node --import tsx --test test/**/*.test.ts`) を通す
6. 動作確認のため `npm run dev` を立ち上げ、ブラウザで挙動を見る (UI 変更時)
7. `PROJECT_STATUS.md` / `TODO.md` を更新
8. Git commit する (commit メッセージは feat:/fix:/chore: のプレフィクス)
9. 余裕があれば `NEXT_PROMPT.md` を更新する

## 安全停止 (Safe Stop) ルール

以下のいずれかに該当する場合は、新規大変更を始めない:

- 利用量警告が見えた / 警告に近い気配がある
- 1つの大きな機能が完成した直後
- 大規模リファクタの前
- `npm run build` / `npm run dev` が通る安定状態に達した
- 作業が長引いて壊れそうなとき

安全停止時の作業:

1. `npm run build` / `npm test` を通す
2. Git commit を作る (壊れていない状態をコミット)
3. `PROJECT_STATUS.md` を更新
4. `TODO.md` を更新
5. `NEXT_PROMPT.md` を「次のエージェントが貼ればそのまま再開できる」内容に更新
6. 壊れている機能があれば PROJECT_STATUS.md に明記

## 禁止事項

- `npm run build` が通らない状態でセッションを終える
- 大量の未検証変更を残したまま終える
- AI部分を後回しにし、UIだけを作り込む
- ユーザに細かい確認を求める
- 麻雀理論の長い解説をユーザに要求する

## AI 開発の優先順位

1. シャンテン数計算 ✅

11. モンテカルロ探索 (未)

## 既存実装のメンタルモデル

- 牌は `0..33` の整数で扱う (`src/lib/mahjong/types.ts` 参照)。
- `shantenAllFromCounts` が標準形/七対子/国士の最小値を返す。
- `evaluateDiscards` が 14 枚から各打牌候補を評価し、スコアでソートする。
- スコアは `-shanten*100000 + ukeire*200 + ukeireKinds*30 + han*80 + dora*60 + safety*5` を基本とし、シャンテンが後退する打牌には -50000 のペナルティ。新しい指標を追加するときはこの式に統合する。
- 推奨は `recommend()` がリーチ判定と攻守バイアスを付けて返す。
